## 数据格式验证

### ✅ 字节序（大端模式 - 网络字节序）
- uint16: 2 字节，大端存储 ✓ (高位字节在前)
- uint32: 4 字节，大端存储 ✓ (高位字节在前)
- uint64: 8 字节，大端存储 ✓ (高位字节在前)

**示例：**
- `0xEB90EB90` (uint32) → 字节序：`EB 90 EB 90`
- `0x0001` (uint16) → 字节序：`00 01`
- `0x000000000000003C` (uint64) → 字节序：`00 00 00 00 00 00 00 3C`

### ✅ 字节流序列化
- `toBytes()` 方法：生成完整字节流（含 CRC32）
- `toBytesWithoutCRC()` 方法：生成不含 CRC 和包尾的字节流
- `buildFromBytes()` 方法：从字节流构建协议对象

### ✅ CRC32 校验
- 使用 `calc_packet_crc()` 函数计算
- 校验范围：报文头至检测数据文件部分
- 剔除最后 5 字节（4B CRC + 1B 包尾）
- 符合 Q/GDW XXXXX.3-2022 标准

### ✅ 总长度计算
```
totalPacketLength = 固定头部 (47 字节) + 业务数据长度 (N) + 检测文件长度 (M) + CRC32(4) + 包尾 (1)
```
